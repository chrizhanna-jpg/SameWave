# Firecircle assets

## Bundled runtime (this repo)

- **Audio stubs:** `../audio/firecircle/*.wav` — generated by `pnpm run gen:firecircle` (see `scripts/gen-firecircle-assets.mjs`). Replace with mastered loops locally; do not commit long licensed stems unless cleared.
- **WebP tiles (`firecircle_<userId>_64.webp` / `_128.webp`):** not committed here. The app loads **remote same-origin** `thumbnailUrl` from `/api/photos/atlas` when the API adds `userId` + `thumbnailUrl` to connections; otherwise tiles use a **local gradient placeholder** (no third-party image upload).

## Preview composite

- Run the app, open **Atlas → Wavefire** with an active cluster, capture the map (e.g. `react-native-view-shot`). Tiles sit in **oversampled map coordinates** and pan/zoom with the globe.

## Manual steps you may still need

1. **Server:** extend `GET /api/photos/atlas` connections with `userId` and same-origin `thumbnailUrl` (or `/path` only) so real faces can appear.
2. **Optional offline WebP pack:** run ImageMagick / `cwebp` per `integration.json` naming; keep processing on your machines or CI — **never** ship raw user bytes to external “processing” services without consent.
3. **Audio:** swap stub WAVs for designed ocean / fire beds; keep levels close to the dB targets in `integration.json`.
4. **iOS pan:** if stereo pan is inaudible, expect to add a native audio module or pre-panned stems.

See **`integration.json`** for machine-readable parameters.
